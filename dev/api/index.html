Python API¶

The Python package exposes one connection type, immutable protocol events, actor roles and states, receive statuses, and protocol errors.

API methods operate on protocol state only. Methods that send HTTP return bytes for the caller to write; receive_data() accepts bytes already read by the caller.

Connection ¶

A Sans-I/O HTTP/1 connection.

Supply transport bytes with receive_data(), poll semantic events with next_event(), and write bytes returned by the send methods. Client and server state are tracked together regardless of the local role.

Construct a connection as:

Connection(
    role,
    *,
    max_head_bytes=65536,
    max_header_count=100,
)

role selects the local client or server endpoint. max_head_bytes defaults to 65,536 and limits each inbound head or trailer section, including an incomplete chunk-size line. max_header_count defaults to 100 and limits each inbound head or trailer section's field count.

Attributes:

Name Type Description
local_state State

The local protocol actor's lifecycle state.
peer_state State

The peer protocol actor's lifecycle state.
peer_http_version bytes | None

The most recently parsed peer HTTP version, if any.
client_is_waiting_for_100_continue bool

Whether the client is waiting for 100 Continue.
trailing_data tuple[bytes, bool]

Bytes retained beyond the HTTP boundary and whether transport EOF was received.

Raises:

Type Description
ValueError

If any inbound limit is zero.

next_event ¶

next_event()

Return the next peer event or a receive status.

Returns:

Name Type Description
event_or_status Request | InformationalResponse | Response | Data | EndOfMessage | ConnectionClosed | ReceiveStatus

A protocol event, ReceiveStatus.NEED_DATA when more transport bytes are required, or ReceiveStatus.PAUSED at a cycle or switch boundary.

Raises:

Type Description
RemoteProtocolError

If peer bytes violate HTTP syntax, framing, configured limits, or protocol state.

Roles and states¶

Role ¶

An HTTP actor role.

Attributes:

Name Type Description
CLIENT Role

The actor that sends requests and receives responses.
SERVER Role

The actor that receives requests and sends responses.

State ¶

An observable HTTP/1 actor lifecycle state.

Attributes:

Name Type Description
IDLE State

No message has started.
SEND_RESPONSE State

The server can send informational or final responses.
SEND_BODY State

The actor can send or receive body data and message end.
DONE State

The actor completed its message for the current cycle.
MIGHT_SWITCH_PROTOCOL State

The request ended while a switch decision is pending.
SWITCHED_PROTOCOL State

HTTP processing ended after a successful switch.
MUST_CLOSE State

The actor completed its message but reuse is forbidden.
CLOSED State

The actor closed its transport side.
ERROR State

A protocol error permanently poisoned the actor.

ReceiveStatus ¶

A non-event result returned by Connection.next_event().

Attributes:

Name Type Description
NEED_DATA ReceiveStatus

Supply more peer bytes before polling again.
PAUSED ReceiveStatus

Complete the active cycle or hand off a switched protocol before polling HTTP again.

A parsed final response head.

Attributes:

Name Type Description

Final status code.

Reason-phrase bytes.

Data ¶

A decoded piece of message body.

chunk_start and chunk_end preserve HTTP chunk boundaries when the peer used chunked transfer coding.

Attributes:

Name Type Description
data bytes

Decoded body bytes.
chunk_start bool

Whether this data begins an HTTP chunk.
chunk_end bool

Whether this data ends an HTTP chunk.

EndOfMessage ¶

The end of one HTTP message, optionally carrying trailer fields.

EndOfMessage() creates an event without trailers. Pass an iterable of (name, value) pairs to construct one with trailers.

Attributes:

Name Type Description
trailers tuple[tuple[bytes, bytes], ...]

Ordered trailer (name, value) byte pairs.

ConnectionClosed ¶

Clean transport EOF from the peer.
